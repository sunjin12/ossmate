# Changelog

All notable changes to Ossmate will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

> From v0.1.0 onwards, this changelog is generated by Ossmate itself via `/release-notes`.

## [Unreleased]

### Added
- Project skeleton (Phase 0)
- Memory templates ([.claude/CLAUDE.md](.claude/CLAUDE.md))
- Minimal `.claude/settings.json` with permissions baseline
- MIT license, README skeleton, .editorconfig
- Two Python packages scaffolded: `mcp/ossmate_mcp`, `cli/ossmate`
- Output styles `maintainer` and `changelog` (Phase 1)
- Cross-platform status line ([.claude/statusline.sh](.claude/statusline.sh) + [.claude/statusline.cmd](.claude/statusline.cmd))
- `outputStyle` and `statusLine` wired in `.claude/settings.json`
- Recommended keybindings sample ([.claude/keybindings.json.example](.claude/keybindings.json.example))
- First slash command [/triage-issue](.claude/commands/triage-issue.md) ‚Äî Bash-only (Phase 2). Classifies an issue, proposes existing labels, drafts a maintainer reply. Read-only `gh` calls; never closes/comments/labels by itself
- All five hook events wired (Phase 3):
  - `PreToolUse(Bash)` ‚Äî [pre_tool_use_guard.py](.claude/hooks/pre_tool_use_guard.py) blocks force-push, push to main, npm/twine publish, gh release create, gh pr merge, gh issue/pr state mutations
  - `PostToolUse(Bash)` ‚Äî [post_tool_use_audit.py](.claude/hooks/post_tool_use_audit.py) appends every Bash call to `.ossmate/audit.jsonl`
  - `UserPromptSubmit` ‚Äî [user_prompt_router.py](.claude/hooks/user_prompt_router.py) auto-fetches GitHub `#1234` references and injects them as additionalContext
  - `SessionStart` ‚Äî [session_start_brief.py](.claude/hooks/session_start_brief.py) injects open PR / new issue / stale / last-tag summary on startup or resume
  - `Stop` ‚Äî [stop_summary.py](.claude/hooks/stop_summary.py) appends one line per session to `.ossmate/journal.md`
- Shared hook helpers in [.claude/hooks/_lib.py](.claude/hooks/_lib.py)
- Hook contract test suite in [tests/test_hooks.py](tests/test_hooks.py) ‚Äî 21 hermetic tests covering all 5 events, malformed input, and re-entry guard
- MCP server `ossmate` (Phase 4) built on FastMCP ‚Äî 11 tools and 3 template resources exposed over stdio:
  - `repo.*` ‚Äî `detect_project_type`, `list_recent_commits` (filesystem + git, no network)
  - `changelog.*` ‚Äî `parse` (Keep-a-Changelog 1.1), `propose_bump` (Conventional Commits ‚Üí semver)
  - `github.*` ‚Äî `list_open_prs`, `list_merged_prs_since`, `get_pr_diff`, `list_stale_issues`, `whoami` (gh CLI shell-out, graceful degradation when `gh` is missing)
  - `deps.*` ‚Äî `read_lockfile` (npm / poetry / uv / cargo auto-detect), `check_advisories` (osv.dev batched query)
  - Resources ‚Äî `templates://release-notes`, `templates://issue-stale-nudge`, `templates://welcome`
- MCP server registered via project-level [.mcp.json](.mcp.json); `enabledMcpjsonServers` gate added to [.claude/settings.json](.claude/settings.json)
- 25 hermetic MCP tests in [tests/test_mcp_tools.py](tests/test_mcp_tools.py) ‚Äî full suite now 46 tests, ~4.6 s
- Six subagents (Phase 5) under [.claude/agents/](.claude/agents/), each pinned to a model tier matching the work's complexity:
  - `haiku` ‚Äî `issue-classifier` (bulk classification, JSON-only output), `community-greeter` (first-contributor welcome drafts)
  - `sonnet` ‚Äî `pr-triager` (single-PR triage with verdict + review draft), `release-notes-writer` (Keep-a-Changelog from merged PRs), `dep-auditor` (OSV.dev advisories via MCP)
  - `opus` ‚Äî `security-reviewer` (deep PR security pass: secrets, injection, auth, supply-chain, CI workflows)
- Seven new skills (Phase 5) wired to delegate to the appropriate subagent via the Task tool: `/triage-pr`, `/release-notes`, `/stale-sweep`, `/onboard-contributor`, `/audit-deps`, `/security-review-pr`, `/changelog-bump`. Skills also enforce policy consistency ‚Äî none allow-list destructive `gh` verbs that would bypass the PreToolUse hook
- 41 hermetic contract tests in [tests/test_agents_skills.py](tests/test_agents_skills.py) ‚Äî validate frontmatter (name/description/model/tools), lock in the model assignment matrix, forbid Write/Edit in subagent tools, forbid destructive verbs in skill `allowed-tools`. Full suite now **87 tests, ~4.4 s**
- Plugin packaging (Phase 6) ‚Äî Ossmate is now installable as a Claude Code plugin:
  - [.claude-plugin/plugin.json](.claude-plugin/plugin.json) ‚Äî plugin manifest pointing at the existing `.claude/commands/`, `.claude/agents/`, `.claude/output-styles/` plus plugin-context hook + MCP configs
  - [.claude-plugin/marketplace.json](.claude-plugin/marketplace.json) ‚Äî single-plugin self-marketplace (GitHub source `sunjin12/ossmate@main`), so users install with `/plugin marketplace add <raw-url>` then `/plugin install ossmate@ossmate`
  - [.claude-plugin/hooks.json](.claude-plugin/hooks.json) ‚Äî twin of the project hooks config but pathed via `${CLAUDE_PLUGIN_ROOT}` so all 5 events keep firing once installed in someone else's repo
  - [.claude-plugin/mcp.json](.claude-plugin/mcp.json) ‚Äî twin of `.mcp.json` reaching the bundled `mcp/ossmate_mcp/src` through `${CLAUDE_PLUGIN_ROOT}`
- 15 hermetic contract tests in [tests/test_plugin_manifest.py](tests/test_plugin_manifest.py) ‚Äî validate manifest JSON, kebab-case names, install-URL resolution (`ossmate@ossmate`), GitHub source descriptor, that all 5 hook events are mirrored, and that no plugin-context file references `${CLAUDE_PROJECT_DIR}` (which would break post-install). Full suite now **102 tests, ~4.5 s**
- Standalone CLI (Phase 7) built on `claude-agent-sdk` + Typer ‚Äî same workflows as the plugin, but for CI / terminals / remote shells where Claude Code isn't running. Eight subcommands (`triage-pr`, `triage-issue`, `release-notes`, `stale-sweep`, `onboard-contributor`, `audit-deps`, `security-review-pr`, `changelog-bump`) map 1:1 to the slash commands by loading the same `.claude/commands/*.md` body ‚Äî edit the skill, both front-ends update at once
  - [cli/ossmate/src/ossmate/prompts.py](cli/ossmate/src/ossmate/prompts.py) ‚Äî minimal frontmatter parser + `$1` / `$ARGUMENTS` placeholder renderer (no PyYAML dep)
  - [cli/ossmate/src/ossmate/agent.py](cli/ossmate/src/ossmate/agent.py) ‚Äî `ClaudeSDKClient` wrapper with lazy SDK import (so `--dry-run` works without the SDK installed) and a `--dry-run` JSON plan printer that surfaces the rendered prompt + `ClaudeAgentOptions` for inspection
  - [cli/ossmate/src/ossmate/tools/repo.py](cli/ossmate/src/ossmate/tools/repo.py) ‚Äî walks upward from cwd to find `.claude/`, then resolves the bundled MCP server's stdio config to absolute paths so the SDK can launch it without a Claude Code env
  - [cli/ossmate/src/ossmate/cli.py](cli/ossmate/src/ossmate/cli.py) ‚Äî Typer app, all subcommands accept `--dry-run`, `--cwd`, `--model` plus skill-specific flags (`--days`, `--label`, `--since`, `--ecosystem`, `--focus`); reconfigures stdout/stderr to UTF-8 on Windows
- 21 hermetic CLI tests in [tests/test_cli.py](tests/test_cli.py) ‚Äî validate skill loading for every `.claude/commands/*.md` file, placeholder substitution corner cases, project-root discovery from nested dirs, MCP config parity with `.mcp.json`, dry-run JSON shape, and (via Typer's `CliRunner`) end-to-end argument plumbing including the invariant that every skill has a CLI subcommand and vice versa. Full suite now **123 tests, ~5.3 s**
